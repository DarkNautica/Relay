HTTP API Reference

All API endpoints are prefixed with /apps/{appId}. Authenticated endpoints require Authorization: Bearer {appSecret}.

Publishing

POST /apps/{appId}/events Requires auth

Publish an event to a single channel.

Request Body

FieldTypeRequiredDescription
channelstringYesTarget channel name
eventstringYesEvent name
datastringYesJSON-encoded event payload
socket_idstringNoExclude this socket from receiving the event

Response

{ "ok": true }

Example

curl -X POST http://localhost:6001/apps/my-app/events \
  -H "Authorization: Bearer my-secret" \
  -H "Content-Type: application/json" \
  -d '{"channel":"chat","event":"message","data":"{\"text\":\"Hello\"}"}'
POST /apps/{appId}/events/batch Requires auth

Publish multiple events in a single request.

Request Body

{
  "batch": [
    { "channel": "chat", "event": "message", "data": "{\"text\":\"Hello\"}" },
    { "channel": "alerts", "event": "notify", "data": "{\"level\":\"info\"}" }
  ]
}

Response

{ "ok": true, "count": 2 }

Example

curl -X POST http://localhost:6001/apps/my-app/events/batch \
  -H "Authorization: Bearer my-secret" \
  -H "Content-Type: application/json" \
  -d '{"batch":[{"channel":"chat","event":"msg","data":"{}"}]}'

Channels

GET /apps/{appId}/channels Requires auth

List all active channels for the app.

Response

{
  "channels": [
    {
      "name": "chat",
      "type": "public",
      "subscriber_count": 5,
      "occupied": true,
      "app_id": "my-app"
    }
  ]
}

Example

curl http://localhost:6001/apps/my-app/channels \
  -H "Authorization: Bearer my-secret"
GET /apps/{appId}/channels/{channelName} Requires auth

Get info for a single channel.

Response

{
  "name": "chat",
  "type": "public",
  "subscriber_count": 5,
  "occupied": true,
  "app_id": "my-app"
}

Example

curl http://localhost:6001/apps/my-app/channels/chat \
  -H "Authorization: Bearer my-secret"
GET /apps/{appId}/channels/{channelName}/users Requires auth

Get the members of a presence channel.

Response

{
  "users": [
    { "id": 1, "user_info": { "name": "Alice" } },
    { "id": 2, "user_info": { "name": "Bob" } }
  ]
}

Example

curl http://localhost:6001/apps/my-app/channels/presence-room/users \
  -H "Authorization: Bearer my-secret"

Authentication

POST /apps/{appId}/auth

Generate an auth token for a private or presence channel. This endpoint does not require Bearer authentication (it's called by the client-side SDK).

Request Body

FieldTypeRequiredDescription
socket_idstringYesThe connecting client's socket ID
channel_namestringYesChannel to authorize
channel_datastringNoJSON user data (required for presence channels)

Response

{
  "auth": "my-key:a1b2c3d4e5f6...",
  "channel_data": "{\"user_id\":1,\"user_info\":{\"name\":\"Alice\"}}"
}

Example

curl -X POST http://localhost:6001/apps/my-app/auth \
  -H "Content-Type: application/json" \
  -d '{"socket_id":"123.456","channel_name":"private-user.1"}'

Other Endpoints

GET /apps/{appId}/channels/{channelName}/events Requires auth

Get event history for a channel. Query param: ?limit=50 (default 50).

GET /apps/{appId}/events/log Requires auth

Get the 20 most recent events for the app (debug log).

GET /stats

Server stats: total connections, channels, per-app counts. No auth required.

GET /health

Health check endpoint. Returns 200 OK if the server is running.

WebSocket Protocol

Clients connect via WebSocket at ws://{host}:{port}/app/{appKey}. All messages are JSON.

Server → Client Events

EventDescription
relay:connection_establishedSent on connect. Contains socket_id.
relay:subscription_succeededChannel subscription confirmed. Presence channels include member list.
relay:member_addedA member joined a presence channel.
relay:member_removedA member left a presence channel.
relay:pongResponse to a ping.
relay:errorError message with code and message.

Client → Server Events

EventDescription
relay:subscribeSubscribe to a channel. Include auth for private/presence.
relay:unsubscribeUnsubscribe from a channel.
relay:pingKeep-alive ping.
client-*Client events (private/presence only). Must start with "client-".
